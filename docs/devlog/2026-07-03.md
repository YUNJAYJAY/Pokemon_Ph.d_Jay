# 2026-07-03: Git 저장소 및 프로젝트 구조 정리

## 1. 진행 상황

### 1.1 Git Repository 생성

포켓몬 박사 챗봇 프로젝트를 위한 Git repository를 생성하였다.

현재 단계는 데이터 수집 단계이다. 우선 PokeAPI와 수동 작성 데이터를 함께 활용하여 포켓몬 기본 정보, 종 정보, 기술 정보, 특성 정보, 타입 상성표 등을 수집하고 정제할 예정이다.

### 1.2 디렉터리 구조 설계

## 2. 디렉터리별 역할

### 2.1 data/raw

`data/raw`는 원본 데이터를 저장하는 디렉터리이다.

이 디렉터리에는 아직 정제되지 않은 데이터를 저장한다.

```text
data/raw/
├── pokeapi/
└── manual/
```

`pokeapi` 디렉터리는 PokeAPI에서 받아온 원본 데이터를 저장하는 공간이다.

`manual` 디렉터리는 직접 작성하거나 수동으로 정리한 원본 데이터를 저장하는 공간이다. 예를 들어 세대별 특수 규칙이나 게임별 공략 정보처럼 API에서 제공하지 않는 정보를 이곳에 저장할 수 있다.

### 2.2 data/processed

`data/processed`는 최종 정제 데이터를 저장하는 디렉터리이다.

이곳에는 챗봇, 규칙 엔진, 웹 서비스에서 직접 사용할 수 있는 형태의 JSON 파일을 저장한다.

| 파일명 | 역할 |
| --- | --- |
| `pokemon.json` | 포켓몬 기본 정보 |
| `pokemon_species.json` | 포켓몬 종 단위 정보 및 도감 설명 |
| `type_chart_gen1.json` | 1세대 타입 상성표 |
| `type_chart_gen2_5.json` | 2~5세대 타입 상성표 |
| `type_chart_gen6_plus.json` | 6세대 이후 타입 상성표 |
| `moves.json` | 기술 정보 |
| `abilities.json` | 특성 정보 |
| `name_map_ko_en.json` | 한국어 이름과 영어 이름 매핑 |
| `battle_rules.json` | 배틀 판정 규칙 |

### 2.3 data/rag_docs

`data/rag_docs`는 LLM이 답변할 때 참고할 문서를 저장하는 디렉터리이다.

RAG 구조에서는 LLM이 직접 모든 정보를 외워서 답변하는 것이 아니라, 관련 문서를 검색한 뒤 그 내용을 바탕으로 답변한다.

| 파일명 | 역할 |
| --- | --- |
| `type_effectiveness_guide.md` | 타입 상성 계산 방법 설명 |
| `generation_changes.md` | 세대별 변경점 설명 |
| `pokemon_profiles.md` | 포켓몬별 설명 문서 |
| `battle_explanation_examples.md` | 배틀 대진 설명 예시 |

### 2.4 data/finetune

`data/finetune`은 QLoRA 파인튜닝에 사용할 학습 데이터를 저장하는 디렉터리이다.

```text
data/finetune/
├── train.jsonl
├── valid.jsonl
└── test.jsonl
```

`jsonl`은 한 줄에 하나의 JSON 객체가 들어가는 형식이다.

각 파일의 역할은 다음과 같다.

| 파일명 | 역할 |
| --- | --- |
| `train.jsonl` | 모델 학습용 데이터 |
| `valid.jsonl` | 학습 중 검증용 데이터 |
| `test.jsonl` | 최종 평가용 데이터 |

### 2.5 scripts

`scripts` 디렉터리는 데이터 수집 및 가공을 위한 Python 스크립트를 저장하는 공간이다.

| 파일명 | 역할 |
| --- | --- |
| `collect_pokeapi.py` | PokeAPI에서 데이터 수집 |
| `build_type_chart.py` | 세대별 타입 상성표 생성 |
| `build_name_map.py` | 한국어-영어 이름 매핑 파일 생성 |
| `make_rag_docs.py` | RAG용 문서 자동 생성 |
| `make_finetune_dataset.py` | QLoRA 학습용 JSONL 데이터셋 생성 |

### 2.6 app

`app` 디렉터리는 실제 서비스 애플리케이션 코드를 저장하는 공간이다.

```text
app/
├── backend/
└── frontend/
```

`backend`는 FastAPI 기반 서버를 구현하는 공간이다. 포켓몬 데이터 검색, 타입 상성 계산, LLM 호출, RAG 검색 등의 기능을 담당한다.

`frontend`는 사용자가 챗봇과 상호작용할 수 있는 웹 화면을 구현하는 공간이다.

## 3. Git 디렉터리 추적 문제

Git은 기본적으로 비어 있는 디렉터리를 추적하지 않는다.

따라서 프로젝트 초기 구조를 유지하기 위해 각 빈 디렉터리에 `.gitkeep` 파일을 생성하였다.

예를 들어 다음과 같은 디렉터리가 비어 있으면 Git에 포함되지 않는다.

```text
data/raw/pokeapi/
data/raw/manual/
data/processed/
data/rag_docs/
data/finetune/
app/backend/
app/frontend/
```

이 문제를 해결하기 위해 각 디렉터리에 `.gitkeep` 파일을 추가하였다.

```text
data/raw/pokeapi/.gitkeep
data/raw/manual/.gitkeep
data/processed/.gitkeep
data/rag_docs/.gitkeep
data/finetune/.gitkeep
app/backend/.gitkeep
app/frontend/.gitkeep
```

이를 통해 실제 데이터나 코드가 아직 없는 상태에서도 GitHub에 프로젝트 디렉터리 구조를 유지한 채 push할 수 있다.
